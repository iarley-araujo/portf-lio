## 📂 Estrutura do Repositório

Este repositório funciona como um *monorepo* (vários projetos dentro de um só), refletindo a estrutura do meu servidor (public_html). Abaixo está a organização das pastas:

```text
📁 portf-lio/
├── 📁 4xia/       # Sistema interno do Aximoz (APIs, Scraper, Simulador e Auth)
├── 📁 axia/       # Assets e mídias do projeto Aximoz Wallet
├── 📁 img/        # Imagens globais e de capa dos projetos
├── 📁 pfci/       # PFCI Saúde (Plataforma de estudos médicos com modelos 3D)
├── 📁 precos/     # Landing pages e tabelas de preços
├── 📁 solutimix/  # Plataforma de e-commerce/serviços (Login, Produtos, PHPMailer)
├── 📄 index.html  # Página inicial do portfólio web
└── 📄 default.php # Configurações padrão do servidor

🌟 Projetos em Destaque
1. PFCI Saúde ⚕️
Uma plataforma interativa voltada para estudos de medicina.

Recursos: Sistema completo de autenticação (Login/Cadastro/Recuperação de senha), upload e gerenciamento de fotos de perfil, e integração de modelos anatômicos em 3D (como o esqueleto.glb) para visualização interativa no navegador.

Tecnologias: PHP, JavaScript, HTML, CSS, MySQL.

2. Aximoz Wallet (4xia/axia) 📈
Um dashboard financeiro inteligente para investidores gerenciarem seus portfólios.

Recursos: Web scraping para buscar dados de ativos em tempo real, simulador de investimentos, operações CRUD para ativos do usuário e sistema seguro de controle de sessão (logout/user actions).

Tecnologias: PHP (APIs RESTful e Scraper), JavaScript (Manipulação do DOM e Fetch), SQL.

3. Solutimix 🛒
Sistema web para simulação de compras e catálogo de produtos.

Recursos: Envio de e-mails transacionais (integrado com PHPMailer), sistema de "esqueci minha senha", painel de contato, vitrine de produtos e rotas seguras de verificação de login.

Tecnologias: PHP orientado a objetos, PHPMailer, HTML/CSS modular.

⚙️ Como rodar o projeto localmente
Como o projeto possui muito código em PHP de back-end (sistemas de login, banco de dados, web scrapers), você precisará de um servidor local (como XAMPP, WAMP ou Laragon) para testar os arquivos.

Faça o clone do repositório:

Bash
git clone [https://github.com/iarley-araujo/portf-lio.git](https://github.com/iarley-araujo/portf-lio.git)
Mova a pasta para o diretório raiz do seu servidor local (ex: C:\xampp\htdocs\).

Inicie os módulos do Apache e MySQL no seu servidor.

Importe os bancos de dados (os arquivos .sql geralmente associados a cada projeto) no seu phpMyAdmin.

Acesse no navegador: http://localhost/portf-lio/.
